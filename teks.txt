Ujian Akhir Semester (UAS) - Tugas Proyek Akhir

Mata Kuliah: Pemrograman Web 2

Ketentuan Umum Proyek
1. Proyek dikerjakan secara Mandiri (Individu) sebagai komponen utama penilaian Ujian
Akhir Semester (UAS).
2. Arsitektur aplikasi wajib menggunakan Decoupled Architecture (Arsitektur Terpisah)
yang memisahkan secara penuh antara Server (Backend) dan Klien (Frontend).
3. Batas akhir pengumpulan proyek adalah tanggal pelaksanaan UAS sesuai Jadwal melalui
tautan repositori GitHub publik.
Spesifikasi Teknologi yang Digunakan
Aplikasi yang dibangun wajib memanfaatkan ekosistem teknologi modern berikut:
● Backend Engine: PHP Framework CodeIgniter 4 (CI4) yang dikonfigurasi murni sebagai
RESTful API Server (Resource Controller).
● Frontend Engine: VueJS 3 dengan ekosistem Vue Router berbasis CDN untuk mengelola
komponen dan navigasi Single Page Application (SPA).

● User Interface (UI) Framework: TailwindCSS via CDN untuk standardisasi tampilan utility-
first yang responsif, rapi, dan modern.

● Data Transfer: Library Axios sebagai agen HTTP Request asynchronous dan
MySQL/MariaDB sebagai basis data utama.
Pilihan Tema Studi Kasus (Pilih Salah Satu)
Mahasiswa diwajibkan memilih salah satu tema aplikasi di bawah ini untuk dikembangkan:
1. Sistem Manajemen Inventaris Barang (E-Inventory):
Mengelola data barang, kategori barang, stok, supplier, dan histori barang masuk/keluar.
2. Sistem Pelaporan Pengaduan Layanan Masyarakat (E-Report):
Mengelola data pelapor, kategori aduan (infrastruktur, keamanan, dll), isi laporan, gambar
bukti, dan status penanganan aduan.
3. Sistem Informasi Rental Buku / Komik Digital (E-Library):
Mengelola data buku, kategori genre, data penulis/penerbit, status peminjaman, dan
anggota.
Spesifikasi & Kriteria Fitur Aplikasi (Soal Proyek)
Setiap mahasiswa wajib membangun sistem informasi dari tema yang dipilih dengan
memenuhi indikator fitur teknis sebagai berikut:
1. Arsitektur Database & Backend API (Bobot 35%)
● Relasi Tabel: Membuat minimal 3 tabel database yang saling berelasi (Contoh: tabel

users, tabel kategori, dan tabel produk/barang).
● RESTful Endpoints: Menyediakan endpoint API yang lengkap menggunakan Resource
Controller CI4 untuk melayani operasi CRUD data master (Metode GET, POST, PUT,
DELETE).
● Server-Side Security (Token): Menerapkan CodeIgniter Filters untuk memproteksi
endpoint manipulasi data (POST, PUT, DELETE). Endpoint hanya dapat diakses jika
melampirkan Authorization Bearer Token yang valid pada HTTP Header.
● Penanganan CORS: Mengonfigurasi CORS Filter global pada Config/Filters.php agar
API dapat menerima request lintas origin dari framework frontend dengan aman tanpa
terblokir browser.
2. Arsitektur Frontend VueJS SPA & TailwindCSS (Bobot 45%)
● Modul Otentikasi (Sistem Login):
○ Membuat halaman komponen Login yang ditenagai oleh fungsi penembakan API
via Axios POST menuju server.
○ Menyimpan tanda pengenal login (isLoggedIn) dan token otentikasi dari server ke
dalam localStorage browser.
○ Menyediakan tombol Logout dinamis yang otomatis menghapus seluruh sesi
token di penyimpanan lokal.
● Manajemen Komponen & Routing:
○ Memisahkan halaman menjadi file komponen modular berbasis arsitektur SPA
(Contoh: Home.js, Login.js, Dashboard.js, Barang.js).
○ Menggunakan Vue Router untuk perpindahan halaman tanpa memuat ulang
browser (No Hard-Reload).
● Client-Side Security (Navigation Guards):
○ Memasang properti meta: { requiresAuth: true } pada rute panel admin.
○ Membuat fungsi router.beforeEach() untuk mencegat pengguna ilegal yang belum
login agar terlempar secara otomatis ke halaman login.
● Otomatisasi Token (Axios Interceptors):
○ Mengonfigurasi Axios Request Interceptor global untuk menyuntikkan token dari
localStorage secara otomatis ke dalam header setiap request data.
○ Mengonfigurasi Axios Response Interceptor untuk menangkap error 401
Unauthorized secara global, memunculkan alert sesi habis, dan menendang
pengguna kembali ke form login.
● Desain UI Modern (TailwindCSS):
○ Seluruh antarmuka komponen wajib didesain menggunakan class utility
TailwindCSS (Form input, tabel data, modal box interaktif, tata letak menu
samping/atas, dan tombol aksi). Tidak diperkenankan menggunakan CSS murni
bergaya tradisional.
Hak Akses Pengguna (User Matrix)
Aplikasi harus memisahkan hak akses halaman menjadi dua kondisi:
● Pengunjung / Public (Tanpa Login): Hanya dapat melihat halaman Beranda (Landing Page)

berisi informasi umum atau ringkasan total data.
● Administrator (Wajib Login): Dapat mengakses Dashboard utama, mengoperasikan form
input, mengedit data, menghapus data master, dan melakukan aktivitas log-out.
